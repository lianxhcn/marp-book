
# Marp Book 写作提示词（精简版）

请协助我撰写一本面向初学者的 `Marp book`。请严格按照以下定位、结构和写作原则展开。

我将在 VScode 中编辑书稿，使用 Quarto 编译，然后通过 GitHub Pages 发布。你可以参考我此前写的这本 online Book 的模式：

- 参考：https://lianxhcn.github.io/quarto_book/
- github: https://github.com/lianxhcn/quarto_book

## 一、定位

这本书不是 Marp 全功能手册，而是一份轻量、实用、循序渐进的入门书。目标读者是：

* 从未用过 Marp 的用户
* 只会最基本功能、希望再学一点常用技巧的用户
* 希望使用 VS Code 制作 Marp Slides，并输出为 PDF 和 HTML 的用户

这本书的核心目标是：

* 让读者尽快开始用 Marp
* 让读者建立信心，觉得“这并不难”
* 让读者掌握最常用、最高频的设定技巧
* 让读者学会借助 AI 自行继续优化和扩展

注意：

* 不要把这本书写成复杂的技术手册
* 不要系统展开高级 CSS、主题开发、低频冷门设定
* 不要介绍 Quarto、GitHub Pages 等写书和发布工具链
* 重点是“师傅引进门，读者可以自行不断优化”

## 二、写作原则

### 1. 第一章必须让读者迅速获得满足感

第一章的任务是在最短时间内让读者看到第一份可预览的 Marp 幻灯片。路径应尽量简单直接：

1. 安装 VS Code
2. 安装 Marp 插件
3. 新建一个 `.md` 文件
4. 输入最小示例
5. 点击 Marp 图标预览
6. 看到第一份幻灯片

即尽快完成：

`安装 VSCode > 安装 Marp 插件 > 点击 Marp 图标 > 写入 marp: true > 生成第一个幻灯片`

### 2. 正文只讲最常用的内容

正文重点放在高频需求上，例如：

* `marp: true`
* `theme`
* `size`
* `paginate`
* `header`
* `footer`
* 页码 `#/#`
* `#`, `##`, `###`, `---`
* `headingDivider`
* 正文字号与标题字号
* 数学公式
* 图片插入与尺寸控制
* 代码块
* 双栏布局
* 小字号注释
* 提示框 / callout 的简化实现
* PDF / HTML 输出中的常见问题
* 如何借助 AI 生成或修改这些内容

### 3. 高级或低频内容不要硬讲

很多低频问题和进阶需求，不需要在正文中系统展开，而应通过：

* 提示词
* FAQs
* 附录索引

来处理。重点不是把高级内容讲完，而是教会读者如何借助 AI 继续做下去。

### 4. AI 辅助是本书最大亮点

除了最基础的 `--- marp: true ---` 模式外，其它设定尽量配合通用提示词，帮助用户借助豆包、DeepSeek、ChatGPT 等 AI 工具生成符合要求的 Marp YAML、CSS 和 Markdown 页面。

重点是教会用户：

* 如何描述想要的页面效果
* 如何要求 AI 保留原模板风格
* 如何要求 AI 只做局部修改
* 如何要求 AI 只返回可直接复制的代码

## 三、全书结构

请按“5 章正文 + 附录”的结构来规划和写作。

### 第 1 章：10 分钟做出第一份 Marp 幻灯片

内容应包括：

* Marp 是什么
* 安装 VS Code
* 安装 Marp 插件
* 新建 `.md` 文件
* 输入最小示例
* 点击 Marp 图标预览
* 导出 PDF / HTML 的最简单方法

这一章必须轻松、简洁、低压力，重点是让读者尽快“跑起来”。

### 第 2 章：最常用的基础设定

内容应包括：

* `theme`
* `size`
* `paginate`
* `header`
* `footer`
* 页码 `#/#`
* `#`, `##`, `###` 与 `---`
* `headingDivider`
* 一份适合课程讲义的基础模板

目标是让读者从“能用”进步到“像一份正式 Slides”。

### 第 3 章：最常见的内容呈现方式

内容应包括：

* 正文字号与标题字号
* 数学公式
* 图片插入与尺寸控制
* 代码块
* 双栏排版
* 小字号注释
* 提示框 / callout 的简化实现

这一章不追求全面，而是讲最有用、最高频的几类页面效果。

### 第 4 章：图片与输出

这一章重点解决新手最容易卡住的问题。

必须重点介绍两种图片处理思路，但有主次之分：

#### 方案 1：本地项目文件夹 + `Fig/`

这是默认推荐方案，应作为主线展开。要明确说明：

* 建立简单 project folder
* 图片统一放在 `Fig/` 文件夹
* Markdown 中用相对路径插图

并强调：

* 对初学者而言，这种方式最简单
* 这种模式下，**以 PDF 输出最合适**
* 如果输出 HTML，则通常不能只发单个 HTML 文件，而应连同图片文件夹一起分享，才能保证图片正常显示

#### 方案 2：图床

只需简要介绍：

* 图床是什么
* 什么时候值得使用
* 为什么它不是初学者默认方案

这一章的结论应明确：

* **初学者默认推荐：本地图片 + `Fig/` + PDF 输出**
* **若有持续在线分享需求，再考虑图床**

### 第 5 章：借助 AI 自我提升

这是全书的方法论核心，也是最大亮点。

重点写：

* 为什么 AI 特别适合帮助生成 Marp 页面和样式
* 如何描述自己想要的页面效果
* 如何要求 AI 保留现有模板风格
* 如何要求 AI 只做局部修改
* 如何让 AI 只返回可直接复制的 Markdown / YAML / CSS
* 如何把低频需求、高级需求交给 AI 处理

这章不要空泛说理，而要给出实用、通用的提示词模板。

## 四、附录的组织方式

附录不要按技术机制分类，而要按“用户想实现的效果”分类。这样更方便读者在网站左上角搜索框中搜索关键词。

附录建议按下面这类逻辑组织：

* 我想调整文字效果

  * 正文字号
  * 标题字号
  * 小字号注释
  * 页码样式
  * `header` / `footer`

* 我想调整分页方式

  * `---`
  * `##` 自动分页
  * `###` 自动分页
  * 临时避免触发自动分页

* 我想插入图片

  * 插入本地图片
  * 调整图片尺寸
  * 背景图
  * 图文混排
  * 图片为什么不显示

* 我想展示代码和结果

  * 普通代码块
  * 调整代码字号
  * 左代码右输出
  * 左代码右图片
  * 长代码页面

* 我想做更好看的版式

  * 双栏布局
  * 提示框
  * 小字备注
  * 封面页

* 我想解决导出和分享问题

  * PDF 输出
  * HTML 输出
  * 本地图片与 HTML 分享
  * 图床的基本思路

* 我想让 AI 帮我做

  * 生成基础模板的提示词
  * 修改已有模板的提示词
  * 生成双栏布局的提示词
  * 调整字号和配色的提示词
  * 处理图片页的提示词
  * 遇到报错时如何提问

### 附录中每个条目的固定结构

每个条目建议统一采用下面这种写法：

* 我想实现什么效果
* 最小写法
* 推荐写法
* 常见问题
* 提示词

## 五、语言风格要求

整本书应当：

* 轻松
* 友好
* 不制造压力
* 有陪伴感
* 强调“先会一点点，再慢慢优化”
* 尽量让读者觉得“这个我也能做到”

不要写成：

* 技术规范手册
* 参数大全
* 面向高手的深度开发文档

整体气质应是：

**让大家心情放松地逐步学会，建立信心和学会如何自我提升最重要。**
